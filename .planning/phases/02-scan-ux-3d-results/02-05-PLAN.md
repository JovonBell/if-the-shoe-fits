    **Replace entire src/app/page.tsx** with the production wizard state machine.

    The component must implement:

    **1. State declarations:**
    ```typescript
    type WizardStep = 'instructions' | 'camera' | 'processing' | 'results'

    // State
    const [step, setStep] = useState<WizardStep>('instructions')
    const [session] = useState(() => new ScanSession())
    const [currentSide, setCurrentSide] = useState<FootSide>('left')
    const [latestResult, setLatestResult] = useState<MeasurementResult | null>(null)
    const [capturedImageData, setCapturedImageData] = useState<ImageData | null>(null)
    const [cameraError, setCameraError] = useState<{ code: string; message: string } | null>(null)
    const [workerReady, setWorkerReady] = useState(false)

    // Refs
    const bridgeRef = useRef<CVWorkerBridge | null>(null)

    // Hooks
    const { videoRef, startCamera, capturePhoto, stopCamera } = useCamera()
    ```

    **2. Worker initialization (same pattern as current test harness):**
    - Create CVWorkerBridge on mount
    - Poll `bridge.isReady` at 100ms intervals
    - Set `workerReady = true` when ready
    - Cleanup: terminate bridge on unmount

    **3. Step transition handlers:**

    `handleStart`:
    - Set step to 'camera'
    - Start camera via `startCamera()`
    - On camera error: set cameraError with user-friendly message and stay on camera step
    - Camera permission denied: set error `{ code: 'CAMERA_DENIED', message: 'Camera access is required. Please allow camera access in your browser settings and refresh.' }`

    `handleCapture`:
    - Set step to 'processing'
    - Clear cameraError
    - Execute capture pipeline:
      1. `capturePhoto()` -> HTMLCanvasElement
      2. `canvas.toBlob()` -> Blob (jpeg, 0.95 quality)
      3. `normalizeOrientation(blob)` -> ImageData
      4. Draw to temp canvas, `resizeImageData(tempCanvas, 1200)` -> ImageData
      5. Store a COPY of the resized ImageData in state BEFORE sending to worker (because transferable makes it unusable after postMessage). Copy via: `new ImageData(new Uint8ClampedArray(resizedImageData.data), resizedImageData.width, resizedImageData.height)`
      6. `bridge.process(resizedImageData, currentSide)` -> ScanResult
    - On success:
      - `session.setResult(currentSide, result.data)`
      - `setLatestResult(result.data)`
      - `stopCamera()` (CRITICAL: stop camera LED — RESEARCH.md Pitfall 3)
      - Set step to 'results'
    - On failure (ScanResult with success=false):
      - Map error codes to user-friendly messages per UI-SPEC Copywriting Contract:
        - `A4_NOT_DETECTED` -> "Paper not found. Make sure your A4 paper is fully visible and on a dark surface."
        - `FOOT_NOT_DETECTED` -> "Foot not found. Ensure your whole foot is on the paper with good lighting."
        - `POOR_LIGHTING` -> "Poor lighting detected. Move to a brighter area and try again."
        - `CALIBRATION_FAILED` -> "Calibration failed. Make sure the entire A4 paper is in frame."
        - `CV_ERROR` (default) -> "Something went wrong. Please retake the photo."
      - Set cameraError with the mapped message
      - Set step back to 'camera' (inline retry — CONTEXT.md decision)
    - On exception (bridge crash, canvas error, etc.):
      - Set cameraError: `{ code: 'CV_ERROR', message: 'Something went wrong. Please retake the photo.' }`
      - Set step back to 'camera'

    `handleScanOtherFoot`:
    - Set `currentSide` to the opposite side: `currentSide === 'left' ? 'right' : 'left'`
    - Clear `latestResult` and `capturedImageData`
    - Set step to 'camera'
    - Start camera via `startCamera()`

    **4. Render:**
    ```tsx
    return (
      <main className="min-h-dvh bg-cream flex flex-col font-body text-dark">
        <StepIndicator currentStep={step} />

        <div className="flex-1 flex flex-col">
          {step === 'instructions' && (
            <InstructionsStep onStart={handleStart} />
          )}
          {step === 'camera' && (
            <CameraStep
              videoRef={videoRef}
              side={currentSide}
              onCapture={handleCapture}
              error={cameraError}
            />
          )}
          {step === 'processing' && (
            <ProcessingStep />
          )}
          {step === 'results' && latestResult && (
            <ResultsStep
              session={session}
              latestResult={latestResult}
              capturedImageData={capturedImageData}
              onScanOtherFoot={handleScanOtherFoot}
            />
          )}
        </div>
      </main>
    )
    ```

    **5. Critical details:**
    - `min-h-dvh` (not min-h-screen) for correct mobile viewport height on iOS Safari with address bar
    - `bg-cream` on main element sets the page background
    - `font-body` and `text-dark` set default body font and text color
    - Camera stream must be stopped when leaving Camera step on success (stopCamera called before transition to results)
    - Worker readiness polling: exact same pattern as Phase 1 test harness (100ms setInterval)
    - ImageData COPY must be stored in state BEFORE calling bridge.process() because the transferable buffer makes the original unusable
    - `handleCapture` should check `!bridgeRef.current || !workerReady` and bail early if not ready
    - The CameraStep `onCapture` prop fires the full pipeline (handleCapture) — CameraStep itself is just the presentational trigger
