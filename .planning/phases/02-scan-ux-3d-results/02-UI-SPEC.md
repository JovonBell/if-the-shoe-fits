## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, inline padding inside badges |
| sm | 8px | Between measurement label and value, compact inner padding |
| md | 16px | Default card padding, section padding on mobile |
| lg | 24px | Between wizard steps in step indicator, card stack gaps |
| xl | 32px | Between major sections on results screen |
| 2xl | 48px | Top/bottom padding on instruction and results screens |
| 3xl | 64px | Page-level vertical breathing room on tall viewports |

Exceptions:
- Touch targets: minimum 44px height for all tappable buttons (ui-ux-pro-max: touch-target-size rule)

- Step indicator dots: 8px diameter with 16px gap (fits 4 steps in 375px viewport)
- 3D canvas height: 300px fixed on mobile (from RESEARCH.md Canvas pattern)
- Contour overlay canvas: max-height 280px (from RESEARCH.md ContourOverlay pattern)

## Interaction Contract

### Wizard Navigation
- Forward navigation only via explicit button taps — no swipe gestures
- No browser back-button between steps (single-page state machine, no route changes)
- Step indicator is display-only, not clickable
- After first foot scan results: secondary CTA "Scan your other foot" (ghost variant) appears alongside primary "Continue" — user choice
- "Scan your other foot" resets to Camera step with `currentSide` updated to opposite foot

### Camera Step
- Video preview fills screen width with `aspect-video` at minimum; expands to fill available height
- Capture button: 72px circle, bg-maroon, centered at 24px from bottom edge
- On capture success: immediately call `stopCamera()` before transition
- On CV error: show inline error banner (bg-red-50 border-red-200 text-destructive) above capture button — do NOT navigate away; stay on CameraStep; show retry copy specific to error type

### Processing Step
- Spinner: pulsing maroon circle or dual-ring; 48px diameter
- Duration: indeterminate (Worker posts message on completion)
- Copy: "Analyzing your foot..." (body 16px, centered)
- No cancel option — processing typically < 3 seconds per RESEARCH.md

### Results Step
- 3D model mounts once and stays mounted for the session duration (prevents iOS WebGL context exhaustion — RESEARCH.md Pitfall 5)
- Auto-rotate begins on mount; user touch interaction stops auto-rotate
- OrbitControls: `enablePan={true}`, `minDistance={0.15}`, `maxDistance={0.6}`, `autoRotateSpeed={1.5}`
- Measurement cards scroll below 3D canvas — 3D canvas does not scroll

### Transitions
- Step transitions: fade-out 150ms → fade-in 150ms (opacity only, no translate — avoids layout shift)
- `prefers-reduced-motion` check: if true, skip animation, render immediately
- Duration-timing: 150-300ms for all micro-interactions (ui-ux-pro-max rule)

## Instruction Step — Mandatory Dark Surface Warning

From STATE.md blocker: "White floor A4 detection failure — must be addressed in Phase 2 UX instructions."

The dark surface warning card must:
- Appear as the FIRST card in the instruction list (before paper placement)
- Use a warning visual style: yellow-50 background, amber-600 border-left, warning SVG icon
- Copy (locked): "Place your A4 paper on a **dark surface**. Carpet, wood, or dark floor only. White or light floors won't work."
- This is the only exception to the cream/maroon color scheme — warning context justifies amber

## Accessibility Contract

All rules from ui-ux-pro-max CRITICAL priority:

| Rule | Implementation |
|------|----------------|
| Color contrast 4.5:1 | maroon #850321 on cream #fffaef = 8.9:1 (passes AAA); dark #1a1a1a on cream = 17.1:1 (passes AAA) |
| Touch targets 44px | All buttons minimum 44px height; camera capture button 72px |
| Focus states | Visible focus ring: `focus-visible:ring-2 focus-visible:ring-maroon focus-visible:ring-offset-2` on all interactive elements |
| aria-labels | Camera capture button (icon-only): `aria-label="Capture photo"`. Step cards: `role="article"`. 3D canvas: `aria-label="Interactive 3D model of your foot"` |
| Keyboard navigation | Tab order matches visual top-to-bottom order; no focus traps |
| prefers-reduced-motion | Step transitions skip animation; 3D model auto-rotate disabled |
| Form labels | No form inputs in Phase 2 (lead capture is Phase 3) |
